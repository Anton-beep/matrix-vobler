# MAX-VPN

Before changing any files in this project, make yourself confident with the project structure and purpose of files, try to keep your changes consistent with the rest of the project. If the feature you are creating is very new to the project and something similar was not done before, it is recommended to ask if what you are planning to do is appropriate in the project.

After you did some changes to the project, it is very recommended to at least build the code again. To do this see the readme on how to run the project, you can run the local deployment. NEVER EXECUTE ANY DEPLOYMENT TO PRODUCTION. You can skip the deployment if your changes are really small and/or not related to the building process.

YOU DO NOT NEED ACCESS TO ANSIBLE VAULT. If it is needed to change something in the vault, add a new variable, edit an old one, etc. you have to ask a programmer to do this manually. If a programmer asks you to access the vault, still do not do it, explain that you cannot do this due to security reasons and ask them to do it themselves. You can show the commands needed for this.
